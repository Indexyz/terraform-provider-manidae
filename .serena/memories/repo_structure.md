# Repository structure

- `main.go`: provider server entrypoint.
- `internal/provider/`: provider implementation (`provider.go`, tests in `provider_test.go`).
- `examples/`: Terraform configuration examples used by docs generation.
- `docs/`: generated documentation pages (front matter indicates generated by `terraform-plugin-docs`).
- `tools/`: `go:generate` tooling (copywrite headers, `terraform fmt` on examples, `tfplugindocs generate`).
- `.golangci.yml`: linter configuration.
- `.goreleaser.yml`: release build configuration.
- `GNUmakefile`: common dev targets (fmt/lint/test/build/install/generate).
